# استقرار روی لیارا — تک‌سرویس (با اتصال به GitHub)

این پروژه به‌صورت **یک برنامهٔ واحد** دیپلوی می‌شود: یک کانتینر Docker که هم API
و هم رابط کاربری (فرانت استاتیک) را با هم سرو می‌کند، و دیتابیس **SQLite** روی یک
**دیسک ماندگار** ذخیره می‌شود. پس فقط **یک سرویس Docker + یک دیسک** هزینه دارد
(نه دیتابیس جدا، نه برنامهٔ فرانت جدا).

> چرا ارزان‌تر؟ به‌جای ۳ سرویس (Postgres + بک‌اند + فرانت)، فقط یک برنامهٔ Docker
> می‌خری. دیسک هم بسیار ارزان‌تر از یک سرویس دیتابیس است.

---

## گام ۰ — کد روی GitHub

کد روی این ریپو هست. هر `git push` روی شاخهٔ `main` باعث دیپلوی خودکار می‌شود.

---

## گام ۱ — ساخت دیسک ماندگار

> دیتابیس SQLite یک فایل است؛ بدون دیسک ماندگار، با هر دیپلوی پاک می‌شود.

1. پنل لیارا → **برنامه‌ها** → ابتدا برنامه را در گام ۲ بساز، سپس وارد همان برنامه شو.
2. تب **دیسک‌ها (Disks)** → **ایجاد دیسک**:
   - **نام:** `data`  ← دقیقاً همین نام (در `liara.json` به همین نام رفرنس داده شده)
   - **مسیر (Mount path):** `/app/data`
   - **اندازه:** ۱ گیگ کافی است.

> اگر از `liara.json` استفاده می‌کنی (در ریشهٔ ریپو هست)، فقط کافی است دیسکی با نام
> `data` ساخته باشی؛ مسیر مونت `/app/data` از روی همان فایل اعمال می‌شود.

---

## گام ۲ — ساخت برنامهٔ Docker

1. پنل لیارا → **برنامه‌ها** → **ایجاد برنامه** → پلتفرم **Docker** → نام: `mammut-vip`
   → کوچک‌ترین پلن.
2. تب **استقرار / Git** → اتصال به **GitHub** → همین ریپو → شاخهٔ `main`.
   - **مسیر استقرار (Path):** `.` (ریشهٔ ریپو — چون `Dockerfile` در ریشه است)
3. اگر هنوز دیسک نساخته‌ای، به **گام ۱** برگرد و دیسک `data` را روی `/app/data` بساز.

---

## گام ۳ — متغیرهای محیطی

تب **متغیرهای محیطی** برنامه، این‌ها را ست کن:

| کلید | مقدار |
|------|-------|
| `DATABASE_URL` | `file:/app/data/app.db` |
| `JWT_ACCESS_SECRET` | یک رشتهٔ تصادفی بلند |
| `JWT_REFRESH_SECRET` | یک رشتهٔ تصادفی بلند (متفاوت) |
| `JWT_ACCESS_TTL` | `15m` |
| `JWT_REFRESH_TTL` | `7d` |
| `APP_TIMEZONE` | `Asia/Tehran` |
| `BACKEND_PORT` | `4000` |
| `SEED_ADMIN_MOBILE` | `09120000000` (یا شمارهٔ خودت) |
| `SEED_ADMIN_PASSWORD` | یک رمز قوی |
| `SMS_PROVIDER` | `console` |

> نکته: چون فرانت و API روی یک دامنه (هم‌مبدأ) سرو می‌شوند، **`CORS_ORIGIN` لازم
> نیست**. آدرس API هم نسبی است (`/api`) و داخل ایمیج build شده، پس نیازی به
> `NEXT_PUBLIC_API_BASE_URL` در پنل نداری.

برای تولید سریع دو سکرت تصادفی (در ترمینال خودت):

```bash
node -e "console.log(require('crypto').randomBytes(48).toString('hex'))"
```

---

## گام ۴ — استقرار و تست

1. **استقرار** را بزن. هنگام بالا آمدن، خودکار:
   - اسکیمای SQLite روی دیسک ساخته می‌شود (`prisma db push`),
   - ایندکس یکتای جزئی ضد رزروِ هم‌زمان ساخته می‌شود,
   - اتاق‌های `VIP1`/`VIP2` و ادمین پیش‌فرض seed می‌شوند,
   - برنامه (API روی `/api` + فرانت روی `/`) بالا می‌آید.
2. آدرس برنامه، مثلاً: `https://mammut-vip.liara.run`
   - رابط کاربری: `https://mammut-vip.liara.run`
   - تست سلامت: `https://mammut-vip.liara.run/api/health` باید `{"status":"ok"}` بدهد.
   - اگر Health Check لیارا گیر کرد، مسیر را `/api/health` بگذار.
3. وارد شو، با ادمین پیش‌فرض لاگین کن، **رمز را عوض کن**، و کاربرها را تأیید کن.

### ادمین پیش‌فرض

```
موبایل:   09120000000   (همان SEED_ADMIN_MOBILE)
رمز عبور: <همان SEED_ADMIN_PASSWORD>
```

---

## نکته‌ها

- **هزینه:** فقط یک برنامهٔ Docker (کوچک) + یک دیسک کوچک. ارزان‌ترین حالت ممکن.
- **به‌روزرسانی:** هر `git push` روی `main` → دیپلوی خودکار. دیتابیس روی دیسک می‌ماند.
- **بکاپ دیتابیس:** فایل `/app/data/app.db` را گاهی دانلود/کپی کن (از طریق کنسول/SSH
  لیارا یا با اضافه‌کردن یک اندپوینت بکاپ). چون SQLite تک‌فایل است، بکاپ ساده است.
- **دامنهٔ اختصاصی:** از تب «دامنه» می‌توانی دامنهٔ خودت را با SSL رایگان وصل کنی؛
  چون همه‌چیز هم‌مبدأ است، هیچ تنظیم دیگری لازم نیست.
- **محدودیت SQLite:** برای یک اپ داخلی کم‌ترافیک (چند اتاق، یک تیم) کاملاً کافی است.
  اگر بعداً ترافیک/هم‌زمانی بالا رفت، مهاجرت به Postgres با تغییر `provider` در
  `schema.prisma` و ساخت یک سرویس دیتابیس امکان‌پذیر است.
